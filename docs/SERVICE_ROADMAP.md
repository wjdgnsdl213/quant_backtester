# 서비스화 로드맵 — 퀀트 백테스트 대시보드

> 2026-07-07 기준 코드베이스 점검 결과.
> "개인 로컬 도구"를 "외부 사용자에게 제공하는 서비스"로 전환할 때 필요한 것,
> 경쟁 서비스 대비 차별화 포인트, 그리고 품질 개선 항목을 정리한다.

## 현재 상태 요약

| 영역 | 상태 |
|---|---|
| 엔진 | FastAPI. 백테스트·최적화·워크포워드·몬테카를로·멀티심볼·비교·AI 생성·시그널 체크 |
| 전략 | 프리셋 5종 + DSL(AI/블록/JSON), 롱/숏, 포지션 사이징, 장중/종가 손절 |
| 검증 장치 | IS/OOS 과적합 진단, 워크포워드, 몬테카를로, 히트맵, 멀티심볼 — 이 도구의 핵심 자산 |
| 데이터 | yfinance(주식)·ccxt/바이낸스(크립토), 종목당 단일 캐시 + 증분 다운로드 |
| 웹 | Next.js 4페이지(랜딩/백테스트/전략/시그널), 고급 모드, 다크모드 |
| 품질 | pytest 99개(look-ahead·미래참조 자동검출 포함), GitHub Actions CI |
| 미검증 | AI 전략 생성 실호출(API 키 없음), 웹 E2E 테스트 없음 |

## 1. 서비스화에 반드시 필요한 것

우선순위 순. ①~④는 외부 공개의 전제 조건이다.

### ① 사용자 계정과 데이터 분리
- 현재 전략 저장소·감시 목록은 **전역 SQLite 하나** — 모든 방문자가 서로의 전략을 보고 지울 수 있다.
- 필요: 인증(이메일 매직링크 또는 OAuth), `strategies`/`watches` 테이블에 `user_id`, 모든 조회에 사용자 필터.
- SQLite → PostgreSQL 전환 권장(동시 쓰기, 백업, 호스팅 편의).

### ② 시세 데이터 소스의 라이선스/안정성
**가장 중요한 리스크.** 현재 데이터 소스는 서비스화에 부적합하다.
- `yfinance`는 야후 비공식 스크래핑 — 상업 서비스 사용은 야후 ToS 위반 소지가 있고, 예고 없이 차단·형식 변경된다.
- 대안: 유료 API(Polygon, Twelve Data, 한국투자증권 OpenAPI, KRX 정보데이터시스템), 크립토는 바이낸스 공식 API(현행 ccxt 유지 가능).
- 데이터 비용이 서비스 원가의 핵심이므로 요금제 설계와 함께 결정해야 한다.

### ③ 법적 고지·컴플라이언스
- 백테스트 결과·시그널은 **투자 자문이 아님**을 모든 결과 화면과 약관에 명시 ("과거 성과는 미래 수익을 보장하지 않습니다").
- 한국에서 시그널 제공이 유사투자자문업 신고 대상에 해당할 수 있는지 검토 필요 — 서비스 형태(정보 제공 vs 추천)에 따라 달라짐.
- 개인정보처리방침, 이용약관.

### ④ 배포 인프라
- 웹: Vercel (현 Next.js 구조 그대로).
- 엔진: 컨테이너 호스팅(Fly.io/Railway/Cloud Run). CORS 허용 오리진을 배포 도메인으로 갱신, `NEXT_PUBLIC_ENGINE_URL` 환경변수 설정.
- 시세 캐시를 컨테이너 로컬 디스크 대신 볼륨 또는 DB로.

### ⑤ 남용 방지·운영
- 무거운 엔드포인트(최적화·워크포워드·멀티심볼) rate limit — 익명 사용자가 300조합 그리드서치를 무한 반복하면 서버가 죽는다.
- AI 생성은 호출당 API 비용 발생 → 사용자별 일일 한도 필수.
- 에러 추적(Sentry), 구조화 로깅, 헬스체크 모니터링.
- 긴 작업의 작업 큐 전환 검토(현재 동기 처리, 워크포워드는 수 초 — 사용자가 늘면 타임아웃·블로킹 이슈).

## 2. 차별화 기능 (경쟁 서비스 대비)

기존 서비스(TradingView 전략 테스터, QuantConnect, 젠포트 등)와 비교했을 때
이 프로젝트의 각도는 **"코드 없이 + 과적합 검증을 기본 탑재"**다. 이를 강화하는 방향:

### 이미 보유한 차별점 (마케팅 포인트)
- **자연어 → 전략**: 코드·수식 없이 문장으로 전략 생성 (경쟁 서비스 대부분 미지원 또는 부가기능).
- **과적합 검증 3종 기본 탑재**: IS/OOS·워크포워드·몬테카를로가 백테스트와 한 화면에. "수익률 한 숫자"만 보여주는 서비스와의 핵심 차별점.
- **검증된 DSL 샌드박스**: 임의 코드 실행이 없어 보안·안정성 우위 (QuantConnect류는 코드 실행 인프라가 필요).

### 추가하면 차별화가 커지는 것
1. **AI 전략 코치** — 백테스트 결과(지표+과적합 판정+거래 분포)를 Claude에게 주고 "왜 이 전략이 약한지, 뭘 바꿔볼지"를 제안받아 원클릭 적용. 자연어 생성과 묶으면 "대화하며 전략을 다듬는" 유일한 서비스가 될 수 있다. (API 키 확보 시 즉시 착수 가능)
2. **시그널 알림 자동화** — 현재 수동 확인인 /signals를 스케줄러(cron)로 자동 실행 + 이메일/텔레그램 푸시. "백테스트 도구"에서 "운용 보조 도구"로 성격이 바뀌는 기능이며 유료화 근거로 가장 자연스럽다.
3. **전략 공유/마켓** — 저장 전략을 공개 링크로 공유(DSL이 JSON이라 안전하게 공유 가능), 리더보드. 단 공유 전략의 과적합 판정을 함께 노출해 "검증 문화"를 차별점으로.
4. **포트폴리오 백테스트** — 여러 전략×여러 종목에 자본을 배분한 포트폴리오 단위 성과(상관관계·분산 효과). 멀티심볼(개별 실행)과 다른 축.
5. **한국 시장 특화** — 국내 서비스로 간다면: KRX 전 종목 검색, 원화 표기, 국내 수수료·세금 프리셋(증권거래세 등). yfinance의 `.KS` 지원은 있으나 종목 검색·한글명이 없다.

## 3. 개선하면 좋은 점 (품질·정확도)

### 백테스트 정확도
- ~~포지션 사이징~~ ✅ (size_pct, 2026-07-07)
- ~~장중 손절/익절~~ ✅ (intrabar, 2026-07-07)
- **숏 비용 미반영**: 주식 대차 수수료, 크립토 선물 펀딩비. 숏 전략 결과가 낙관적.
- **ATR 기반 사이징/손절**: 고정 %가 아닌 변동성 연동 (지표는 이미 있음, risk 필드 확장만 필요).
- **부분 청산·피라미딩**: 현재 전량 진입/전량 청산만.
- **배당 재투자**: auto_adjust로 수정주가는 반영되나 배당 현금흐름은 미반영.

### 데이터
- 인트라데이(1h) 주식 데이터는 yfinance가 최근 ~730일만 제공 — UI에 한계 안내 없음.
- 상장폐지 종목 미포함 → **생존 편향** 존재. 문서에 명시할 것.
- 거래정지·액면분할 등 이벤트 처리 검증 부족.

### 웹 품질
- **E2E 테스트 0개**: Playwright로 핵심 플로우(백테스트 실행→결과 표시, 전략 저장→비교) 커버 필요. UI를 크게 바꿀 때마다 수동 확인에 의존하는 상태.
- 모바일 실기기 확인 안 됨 (반응형 클래스는 적용돼 있음).
- 로딩 스켈레톤 없음 (버튼 텍스트 변경만).
- 결과 상태가 페이지 이동 시 사라짐 (전역 상태/URL 파라미터로 보존 검토).

### 엔진 구조
- 시그널 계산의 봉 단위 파이썬 루프 — 현재 속도는 충분하나, 분봉 수만 개·동시 사용자 확대 시 numba/벡터화 재검토.
- 워크포워드·최적화의 동기 실행 — 서비스화 시 작업 큐(celery/arq) + 진행률 표시.
- 한글 경로에서 uvicorn `--reload` 미동작 (개발 편의 이슈).

## 4. 제안 로드맵

| 단계 | 내용 | 성격 |
|---|---|---|
| Phase 1 (지금) | AI 키 확보 → 생성 품질 검증 + AI 코치 프로토타입, E2E 테스트, 시그널 스케줄러+알림 | 로컬에서 완성도 올리기 |
| Phase 2 (공개 준비) | 데이터 소스 교체 결정, 인증+사용자별 저장(PostgreSQL), rate limit, 법적 고지 | 서비스 전제 조건 |
| Phase 3 (공개) | Vercel+컨테이너 배포, 모니터링, 요금제(무료: 백테스트 / 유료: AI·알림·대량 최적화) | 런칭 |
| Phase 4 (성장) | 전략 공유/리더보드, 포트폴리오 백테스트, 한국 시장 특화 | 차별화 확장 |
