## A. 서비스하기 위해 반드시 필요한 기능

### A-1. 계정과 데이터 격리
- 🔴 **사용자 인증** — 이메일 매직링크 또는 OAuth(구글/카카오). 세션 관리, 로그아웃.
- 🔴 **사용자별 데이터 분리** — 현재 `storage/strategies.db` 하나를 전원이 공유. `strategies`·`watches`에 `user_id` 추가, 모든 CRUD에 소유자 검증. 지금 구조로 공개하면 방문자끼리 전략을 보고 지울 수 있다.
- 🔴 **SQLite → PostgreSQL** — 동시 쓰기, 마이그레이션, 관리형 백업. SQLite는 단일 사용자 전제.
- 🟡 **계정 삭제·데이터 내보내기** — 개인정보보호법 대응(잊힐 권리).

### A-2. 시세 데이터 (최대 리스크)
- 🔴 **yfinance 교체 또는 이중화** — 야후 비공식 스크래핑이라 상업 서비스에서 ToS 위반 소지 + 예고 없는 차단 리스크. 후보: Polygon, Twelve Data, Alpha Vantage(주식), 한국투자증권 OpenAPI·KRX(국내), 바이낸스 공식(크립토, 현행 ccxt 유지 가능).
- 🔴 **데이터 비용 모델링** — API 요금이 서비스 원가의 핵심. 요금제 설계와 동시 결정.
- 🟡 **데이터 장애 대응** — 소스 다운 시 캐시 폴백 + 사용자 안내(현재는 502 한 줄).
- 🟢 **데이터 소스 추상화 계층** — `loader.py`에 provider 인터페이스를 둬서 소스 교체·추가(미국+한국+크립토 혼합)를 쉽게.

### A-3. 보안 (점검에서 발견된 실제 상태)
- 🔴 **엔진 인증 없음** — `main.py`의 모든 엔드포인트가 무인증. 배포 시 API 키/세션 토큰 미들웨어 필수.
- 🔴 **CORS 하드코딩** — `main.py:41` `allow_origins=["http://localhost:3000", ...]`. 배포 도메인 환경변수화.
- 🔴 **HTTPS** — 리버스 프록시(호스팅 기본 제공) 뒤에 배치.
- 🟡 **DSL 중첩 깊이 명시 제한** — 2,000중첩 not 체인 실험 결과 pydantic 자체 재귀 가드가 ValidationError로 막아주지만, 의도된 방어가 아니므로 스키마에 명시적 깊이 제한(예: 5단계) 권장.
- 🟡 **요청 크기 제한** — 대형 JSON 바디 제한(프록시 레벨).
- 🟢 **의존성 취약점 스캔** — dependabot / `pip-audit` CI 단계.

### A-4. 남용 방지·성능
- 🔴 **Rate limiting** — 무거운 엔드포인트(/optimize 300조합, /walkforward, /multisymbol 8종목, /ai/generate)를 익명으로 무한 호출 가능. IP/사용자별 제한.
- 🔴 **AI 호출 한도** — /ai/generate는 호출당 실비 발생. 사용자별 일일 쿼터 + 요금제 연동.
- 🟡 **작업 큐 전환** — 현재 전부 동기 처리(FastAPI threadpool). 동시 사용자가 늘면 무거운 요청이 스레드를 점유. celery/arq + 진행률 폴링/SSE.
- 🟡 **시그널 체크 병렬화** — `signals.check_watch`가 감시 항목을 순차 처리. 50개 등록 시 심볼별 네트워크 대기가 누적된다. asyncio/스레드풀 병렬화.
- 🟡 **실패 심볼 부정 캐싱** — 존재하지 않는 티커를 반복 조회하면 매번 외부 API를 때린다. 실패 결과 단기 캐시.
- 🟢 **시세 캐시 상한** — `data/cache/`가 무한 성장. LRU 정리 또는 용량 상한.

### A-5. 법률·컴플라이언스
- 🔴 **투자 조언 아님 고지** — 모든 결과 화면·시그널 페이지·약관에 명시. 시그널 페이지에 1차 문구는 넣었으나 전 화면 일관 적용 필요.
- 🔴 **유사투자자문업 검토** — 한국에서 시그널 제공 형태에 따라 신고 대상 여부가 갈림. 공개 전 법률 자문.
- 🔴 **이용약관·개인정보처리방침**.
- 🟡 **생존 편향 고지** — 상장폐지 종목이 데이터에 없어 백테스트가 낙관 편향됨을 문서·UI에 명시.

### A-6. 운영 인프라
- 🔴 **배포** — 웹: Vercel. 엔진: 컨테이너(Fly.io/Railway/Cloud Run) + 볼륨(캐시·DB). `NEXT_PUBLIC_ENGINE_URL` 주입.
- 🔴 **에러 추적** — Sentry(웹+엔진). 현재 엔진 로그는 stdout뿐.
- 🟡 **모니터링·알림** — 헬스체크, 응답시간, 데이터 소스 실패율 대시보드.
- 🟡 **DB·전략 데이터 백업** — 현재 백업 없음.
- 🟡 **상태 페이지** — 장애 시 사용자 안내.
- 🟢 **관리자 대시보드** — 가입자·사용량·AI 비용 집계.

### A-7. 비즈니스 기능
- 🟡 **요금제·결제** — Stripe(글로벌)/토스페이먼츠(국내). 자연스러운 경계: 무료=백테스트+프리셋, 유료=AI 생성·코치, 시그널 자동 알림, 대량 최적화, 멀티심볼 확대.
- 🟡 **이메일 인프라** — 가입 인증, 알림 발송(Resend/SES).
- 🟡 **온보딩** — 첫 방문 시 예제 전략으로 결과를 즉시 보여주는 가이드 투어. 현재는 빈 화면에서 시작.
- 🟢 **사용 분석** — 이벤트 추적(어떤 기능이 쓰이는지)으로 개선 우선순위 결정.
- 🟢 **고객지원 채널** — 문의 폼 또는 디스코드.

## B. 차별화 기능 (추가하면 좋은 것)

기존 보유 차별점: **자연어 전략 생성 + 과적합 검증 3종(IS/OOS·워크포워드·몬테카를로) 기본 탑재 + 코드 실행 없는 안전한 DSL**. 아래는 이를 증폭하는 순서로 나열.

### B-1. AI를 축으로 (보유 자산 증폭)
1. 🟡 **AI 전략 코치** — 백테스트 결과(지표+과적합 판정+거래 분포)를 통째로 주고 "이 전략의 약점과 수정안"을 받아 원클릭 적용. 자연어 생성과 묶으면 "대화하면서 전략을 다듬는" 경험은 현재 시장에 없다. (API 키만 있으면 즉시 착수 가능한 구조)
2. 🟢 **AI 결과 해설** — 지표를 모르는 초보에게 "샤프 0.6은 이런 뜻이고, 이 전략은 하락장에서 약합니다"를 자연어로.
3. 🟢 **자연어 스크리닝** — "최근 한 달 20일선 위에 있는 코스피 종목 찾아줘" (데이터 소스 확장 후).

### B-2. 검증·신뢰를 축으로 (철학의 제품화)
4. 🟡 **전략 신뢰 점수** — IS/OOS·워크포워드·몬테카를로·멀티심볼 결과를 하나의 종합 점수(예: A~F 등급)로. "수익률 300%!"가 아니라 "신뢰 등급 B"로 말하는 서비스라는 정체성.
5. 🟡 **페이퍼 트레이딩(모의 추적)** — 저장 전략을 "추적 시작"하면 그 시점 이후 시그널대로 가상 매매를 기록해 **진짜 아웃오브샘플 성적표**를 쌓는다. 백테스트→실전 사이의 신뢰 격차를 메우는 킬러 기능이고, 시그널 체크 인프라를 이미 갖춰 기반 작업이 적다.
6. 🟢 **실험 노트/전략 버전 관리** — 파라미터를 바꿔가며 실험한 이력을 자동 기록(어떤 조합을 이미 시험했는지). 과적합의 주범인 "될 때까지 돌리기"를 가시화.

### B-3. 운용 도구로 확장
7. 🟡 **시그널 알림 자동화** — /signals를 스케줄러로 매일 자동 실행 + 텔레그램/이메일 푸시. 수동 확인→자동 알림은 유료화 근거로 가장 자연스럽다.
8. 🟢 **포트폴리오 백테스트** — 여러 전략×종목에 자본 배분, 상관관계·분산 효과, 주기적 리밸런싱 시뮬레이션.
9. 🟢 **브로커 연동 자동매매** — 장기 과제(규제·리스크 큼). 페이퍼 트레이딩으로 수요 검증 후.

### B-4. 커뮤니티·콘텐츠
10. 🟢 **전략 공유/포크** — DSL이 JSON이라 코드 실행 위험 없이 안전하게 공유 가능(구조적 이점). 공유 카드에 신뢰 점수를 함께 노출해 "검증 문화"를 차별점으로.
11. 🟢 **OOS 리더보드/대회** — 공개 시점 이후 성과(진짜 미래 데이터)로만 순위를 매기는 대회. 백테스트 과시가 아니라 실전 검증 경쟁.
12. 🟢 **템플릿 갤러리·교육 모드** — 유명 전략(터틀, 듀얼모멘텀 등) 템플릿 + "과적합이 왜 위험한가"를 직접 체험시키는 인터랙티브 튜토리얼.
13. 🟢 **리포트 공유 이미지** — 백테스트 결과를 SNS 공유용 카드 이미지/PDF로 내보내기(신뢰 등급 포함) → 자연 유입 채널.

### B-5. 분석 폭 확장
14. 🟢 **지표 확장** — 스토캐스틱, OBV, MFI, 이치모쿠, VWAP 등. 레지스트리 구조상 지표 하나 추가하면 엔진·UI·AI가 동시 인식(설계 이점 활용).
15. 🟢 **멀티 타임프레임 조건** — "주봉 추세가 상승일 때만 일봉 진입" (DSL 확장 필요, 수요 높음).
16. 🟢 **시즌성·요일 효과 분석**, **상관관계 매트릭스**, **벤치마크 선택**(SPY/KOSPI/BTC 대비 알파).
17. 🟢 **한국 시장 특화** — KRX 전 종목 한글 검색, 증권거래세·수수료 프리셋, 원화 표기. 국내 타겟이면 최우선 차별점.
18. 🟢 **개발자 API** — 엔진 API를 키 발급으로 외부 제공(상위 요금제).

## C. 개선하면 좋은 점

### C-1. 백테스트 정확도 (완료/잔여)
- ✅ 포지션 사이징(size_pct), 장중 손절/익절(intrabar), 숏 지원, look-ahead 방지, 돈치안 당일봉 제외 — 완료
- 🟡 **숏 보유 비용** — 주식 대차수수료·크립토 펀딩비 미반영, 숏 결과 낙관적.
- 🟡 **ATR 연동 손절/사이징** — 고정 %가 아닌 변동성 기반(지표는 이미 있음, risk 필드 확장만).
- 🟢 **부분 진입/청산·피라미딩** — 현재 전량 진입/전량 청산만.
- 🟢 **트레일링 스탑** — 수요 높은 리스크 관리 방식.
- 🟢 **배당 현금흐름** — 수정주가엔 반영되나 배당 재투자 흐름은 미모델링.
- 🟢 **체결 모델 옵션** — 현재 "다음 봉 시가" 고정. 종가 체결/VWAP 근사 등 선택지.

### C-2. 데이터 품질
- 🟡 **생존 편향** — 상폐 종목 부재. 최소한 UI 고지, 장기적으론 상폐 포함 데이터.
- 🟡 **인트라데이 한계 안내** — yfinance 1h는 최근 ~730일만 제공되는데 UI에 안내 없음(그냥 데이터 부족 에러).
- 🟢 **타임존 일관성** — 주식(naive 로컬)과 크립토(UTC) 인덱스 기준이 다름. 시그널 "오늘" 판정이 소스별로 어긋날 수 있음.
- 🟢 **캐시 무결성** — CSV 손상 시 감지·재다운로드 로직 없음.
- 🟢 **분할·거래정지 이벤트 검증** — auto_adjust 신뢰 외 자체 검증 없음.

### C-3. 엔진 구조
- 🟡 **긴 작업 취소·진행률** — 최적화/워크포워드 실행 중 취소 불가, 진행률 없음.
- 🟢 **시그널 계산 성능** — 봉 단위 파이썬 루프. 일봉 수천 개엔 충분하나 분봉·대량 그리드에선 numba/벡터화 재검토.
- 🟢 **결과 캐싱** — 동일 요청(전략+종목+기간) 반복 시 재계산. 해시 기반 결과 캐시.
- 🟢 **API 버저닝** — 외부 공개 시 /v1 프리픽스.

### C-4. 웹 UX
- 🟡 **결과 상태 소실** — 페이지 이동하면 백테스트 결과가 사라짐. 전역 상태 또는 URL 파라미터(공유 가능한 결과 링크로 발전 가능).
- 🟡 **온보딩 빈 화면** — 첫 방문 시 안내문뿐. 예제 결과 프리로드 또는 "예제 실행" 버튼.
- 🟡 **모바일 실기기 점검** — 반응형 클래스는 있으나 실검증 안 됨. 특히 테이블·히트맵 가로 스크롤.
- 🟢 **로딩 스켈레톤** — 현재 버튼 텍스트 변경뿐. 차트 영역 스켈레톤.
- 🟢 **에러 바운더리** — React error boundary 없음. 컴포넌트 오류 시 백화면.
- 🟢 **접근성** — aria-label 부분 적용. 키보드 내비게이션·색약 대비 점검.
- 🟢 **페이지별 SEO 메타** — layout 전역 메타뿐. 라우트별 title/description.
- 🟢 **결과 히스토리** — 이번 세션에서 돌린 백테스트들 목록·재열람.
- 🟢 **i18n** — 한국어 하드코딩. 글로벌 타겟이면 영어 분리.

### C-5. 테스트·품질
- 🟡 **웹 E2E 0개** — Playwright로 핵심 플로우(백테스트 실행→결과, 저장→비교, 시그널 등록→확인). UI 개편 때마다 수동 확인 의존 중.
- 🟡 **API 테스트 커버리지** — /walkforward·/multisymbol·/watches·/signals는 스모크 테스트 없음(단위 테스트만).
- 🟢 **수치 골든 테스트** — 알려진 데이터+전략의 기대 수익률 스냅숏 고정(리팩토링 회귀 방지).
- 🟢 **프로퍼티 테스트** — hypothesis로 임의 DSL/시세에 불변식(자산>0, 포지션∈[-1,1]) 검증.
- 🟢 **부하 테스트** — 동시 요청 시 스레드풀 한계 측정.
